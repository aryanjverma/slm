# College Board Permission Checkpoint

> This checkpoint governs production of the separately verified golden/dev/final artifacts. At
> the user's explicit direction, the current model-evaluation notebook instead reads the existing
> local `artifacts/data/eval_cb_cases.jsonl` file directly and reports its audit findings as
> warnings rather than execution gates.

Status: **unresolved** as of July 10, 2026.

The repository contains legacy, locally derived AP Central artifacts, but v2 does not treat them
as authorized training data or as a publishable golden corpus. College Board's educator terms
require express written permission for reuse, scraping, data mining, transformation, and
redistribution.

Before any official v2 evaluation artifact can be emitted:

1. submit the College Board permission request described in
   `docs/college-board-data-source-research.md`;
2. save the resulting private authorization as `config/college_board_permission.json` using
   `config/college_board_permission.example.json` as the schema;
3. ensure `allowed_uses` contains `evaluation` (and only add `training` or `redistribution` if
   those uses are explicitly granted);
4. complete the named row-by-row review log generated by
   `python scripts/build_golden_v2.py --create-review-template`;
5. run `python scripts/build_golden_v2.py` and retain its audit report and
   `artifacts/data/v2/eval_cb_manifest_v2.json`.

The gated build writes the 53-row combined file, the 27-row `set1` official dev file, and the
26-row `set2` final holdout. It rejects prompt overlap and records portable SHA-256 hashes and
row counts for all three. The final holdout must not be used for tuning.

The private permission record is gitignored. `--audit-only` may diagnose legacy local rows, but it
does not produce or authorize a derived corpus.
